# User Journey & Information Architecture

## User Journey
1. User arrives with intent to build a hardware product
2. Selects preferred city
3. Answers brief scoping questions
4. Receives recommendation (freelancer vs cluster)
5. Explores vetted profiles
6. Reviews trust signals
7. Takes next step (proposal, call, availability)

## Information Architecture
- Landing
- City Selection
- Recommendation
- Listings
- Profile Detail
- Commitment Actions
